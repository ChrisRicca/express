Views

The Views plugin is required by core, and handles the rendering of content such as HTML, and CSS and provides the Request object with the methods Request#render() and Request#partial() .

Naming Views

Currently view names must conform to the following format: NAME.ENGINE.TYPE where NAME represents the arbtrary name of the file such as "user", ENGINE represents the template engine such as "haml", or "sass", and finally TYPE represents the mime type such as "html", "css", which the View module uses to automatically assign the Content-Type header.
